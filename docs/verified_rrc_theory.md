# Verified-RRC 正确性理论草案

## 1. 结论边界

本文件只讨论 Yan 与 Murawaki, *Efficient Provably Secure Linguistic Steganography via
Range Coding*, ACL 2026 中 Algorithm 3/4 的区间与旋转逻辑。规范来源：
<https://aclanthology.org/2026.acl-long.39/>。

严格区分以下结论：

| 结论 | 当前状态 |
|---|---|
| 论文局部终止条件足以保证精确解码 | 被一个精确有理数反例否定 |
| Verified-RRC 一旦停止，Algorithm 4 精确恢复消息 | 已证明，条件为精确算术和确定性重放 |
| Verified-RRC 对所有模型和消息必然有限终止 | 尚未证明，不应声称 |
| 理想连续随机偏移下，token 联合分布等于目标自回归分布 | 在每步偏移条件独立均匀时成立 |
| 256-bit HMAC 偏移与理想连续偏移完全相同 | 不成立；只能在理想随机函数/PRF 假设下给出离散误差或计算安全结论 |
| 未量化 top-p 条件分布的反向 KL `KL(Q_t||P_t)` | 精确等于 `-log(Z_t)` |
| 论文方向的正向 KL `KL(P_t||Q_t)` | `Z_t<1` 时因支持缺失为无穷大 |
| 当前 top-p/量化实现相对完整 LM 的反向 KL仍为 `-log(Z_t)` | 不成立；还要加量化项 `KL(R_t || Q_t)` |
| 每步条件分布匹配可直接推出变长文本分布匹配 | 不成立；数据相关停止会引入长度与选择偏差 |

## 2. 形式化定义

消息长度为 `l`，令 `M=2^l`，秘密整数 `d^{-1}=m` 属于
`{0,...,M-1}`。初始区间为

```text
I^{-1} = [L^{-1}, R^{-1}) = [0, M).
```

在第 `t` 步，父区间宽度为 `Delta^{t-1}=R^{t-1}-L^{t-1}`。偏移
`o_t` 属于 `[0,1)`，旋转映射定义为

```text
T_t(x) = L^{t-1} + ((x-L^{t-1}) + o_t Delta^{t-1}) mod Delta^{t-1}.
```

语言模型概率把父区间分割为半开 token 区间。编码器选择唯一包含
`d^t=T_t(d^{t-1})` 的区间 `I^t=[L^t,R^t)`。

解码器从最终中点 `y^T=(L^T+R^T)/2` 开始，依次应用

```text
T_t^{-1}(y) = L^{t-1} + ((y-L^{t-1}) - o_t Delta^{t-1}) mod Delta^{t-1}
```

得到 `y^{-1}`，最后输出 `round_half_down(y^{-1})`。

## 3. 基本引理

### 引理 1：旋转是双射

对任意半开区间 `I=[L,R)` 和 `o in [0,1)`，`T` 是从 `I` 到自身的双射，且上述
`T^{-1}` 满足 `T^{-1}(T(x))=x`。

证明：平移到 `[0,Delta)` 后，映射等于圆群上的加法
`z -> (z+o Delta) mod Delta`。其逆为加上 `-o Delta`。再平移回 `I` 即得。

### 引理 2：理想偏移下的均匀性

固定任意 `d^{t-1}`。若 `o_t` 独立且服从 `U[0,1)`，则
`d^t=T_t(d^{t-1})` 在 `I^{t-1}` 上连续均匀。

证明：`o_t Delta^{t-1}` 在 `[0,Delta^{t-1})` 上均匀。圆群上把均匀变量平移任意常数
仍为均匀变量。

### 定理 1：条件 token 分布与联合分布

若 token `i` 的子区间长度为 `p_i Delta^{t-1}`，则选择该 token 的概率为 `p_i`。

更严格地，令 `F_{t-1}` 包含截至第 `t-1` 步的全部偏移、秘密轨迹和 token 历史。只要
`o_t` 在给定 `F_{t-1}` 后仍独立服从 `U[0,1)`，引理 2 给出

```text
Pr(S_t=i | F_{t-1}) = q_t(i | S_{<t}).
```

对 `F_{t-1}` 使用全期望公式，同一等式也在只给定 token 前缀 `S_{<t}` 时成立。再由概率
链式法则，对任意有限 token 序列 `s_0,...,s_T`，

```text
Pr(S_0=s_0,...,S_T=s_T)
  = product_{t=0}^T q_t(s_t | s_{<t}).
```

所以理想连续偏移模型匹配的是算法实际分割区间所用的自回归目标分布 `Q`。该定理不自动
把 `Q` 等同于未截断、未改温的原始语言模型分布 `P`。

### 推论 1：有限随机网格的单步偏差

实际 PRF 输出是 `N=2^256` 个离散点之一，不是连续均匀实数。在理想随机函数模型中，固定
历史后令偏移在 `{0,1/N,...,(N-1)/N}` 上均匀。任意长度为 `q_i` 的圆周半开弧包含的网格点
数只能是 `floor(N q_i)` 或 `ceil(N q_i)`，因此

```text
|Pr(S_t=i | F_{t-1}) - q_i| <= 1/N.
```

若该步有 `K_t` 个候选，则单步总变差距离满足 `TV <= K_t/(2N)`。这是理想离散随机函数的
统计界，不是 HMAC-SHA256 的无条件信息论证明。对真实 HMAC，只能在随机密钥和标准 PRF
安全假设下主张其与理想离散随机函数计算不可区分；固定 key 时整个输出序列是确定性的。

### 定理 2：KL 方向、截断与概率量化

令完整分布为 `P=(p_i)`，保留支持为 `S`，保留质量

```text
Z = sum_{i in S} p_i > 0,
Q_i = p_i/Z  (i in S).
```

则未量化条件分布的**反向 KL** 满足

```text
KL(Q || P) = sum_{i in S} Q_i log(Q_i/p_i) = -log Z.
```

但论文第 2.3、2.4、6.2 节使用的安全方向是 `KL(P_cover || P_stego)`。若把完整 softmax `P`
作为 cover、把 top-p 条件分布 `Q` 作为 stego，且 `Z<1`，则存在 `i not in S` 使
`P_i>0,Q_i=0`，所以

```text
KL(P || Q) = +infinity.
```

因此 `-log Z` 不能称为论文方向的截断 KL。它只是 `KL(Q||P)`。要保留理想零 KL，实验必须
把使用完全相同 temperature、top-k 和 top-p 的正常生成分布 `Q` 定义为 cover，然后比较
stego 实际分布与该 `Q`。

若实现量化并重归一化后实际使用分布 `R`，且 `R` 的支持包含于 `S`，则

```text
KL(R || P) = -log Z + KL(R || Q).
```

证明只需代入 `p_i=Z Q_i`。因此 `DistributionSnapshot.truncation_kl_nats` 记录的是反向 KL
截断项，不包含 `probability_quantum` 引入的非负量化项。若 cover 已定义为同配置的 `Q`，
论文方向应报告 `KL(Q||R)`；它通常不等于 `KL(R||Q)`。若熵救援改变了 temperature，则还要
明确 cover 是否同步采用相同控制器；相对原始固定温度基线的 KL 不能由 `-log Z` 推出。

### 定理 3：逐步匹配不推出停止文本匹配

考虑独立公平 bit 流。stego 在首 bit 为 0 时于长度 1 停止，否则生成第二个 bit 后停止。
在每个尚未停止的步骤，下一个 bit 条件分布都严格是公平分布。stego 的完整序列分布为

```text
Pr(0)=1/2,  Pr(10)=1/4,  Pr(11)=1/4.
```

现在令 cover 先独立采样同样的长度边缘分布
`Pr(L=1)=Pr(L=2)=1/2`，再生成 `L` 个公平 bit。此时

```text
Pr_cover(0)=1/4,
```

事件“输出恰为 0”的区分优势为 `1/4`，两种完整序列分布的总变差距离精确为 `1/2`。所以
即使活动步骤条件分布和长度边缘分布都匹配，数据相关停止仍会把 token 内容与长度相关联。
`stopped_process_counterexample` 用 `Fraction` 精确复现该结论。

一般地，stego 序列概率包含历史相关停止 hazard：

```text
Pr_stego(s_0,...,s_T,T)
  = product_t q_t(s_t|s_<t) * Pr(T | s_0,...,s_T).
```

只比较 token 乘积会漏掉最后一项。论文 Appendix B.3 Step 2 把随机停止时间 `T` 与固定
`L=E[T]` 的正常生成比较，二者甚至不一定在同一长度样本空间；“停止状态对观察者不可见”
也不能消除可见的输出长度和长度条件内容偏差。

一个充分修复是使用公开固定总长度 `N`：嵌入完成后不立即发送，而是继续从与 cover 完全
相同的 `Q_t` 采样到 `N`。若嵌入阶段和填充阶段在每一步给定公开历史的 token 条件分布都
是 `Q_t`，则由链式法则，固定长度全文分布等于 cover。句号收尾或“生成到自然结束”仍是
数据相关停止，不能自动获得该结论。

项目中的初版实现见 [`fixed_length_rrc.md`](fixed_length_rrc.md)。实现实际匹配的是概率量化后
分布 `R_t`，并使用认证标签让接收端在不公开真实停止时间的情况下扫描恢复前缀。若只发布
成功完成的样本，输出会被成功事件条件化；因此论文必须同时报告固定预算下的 fallback 率，
不能删除失败样本后继续声称无条件全文分布匹配。

## 4. 论文终止条件的精确反例

论文局部条件是

```text
-1/2 < midpoint(I^t) - d^t <= 1/2.
```

以下两步反例全部使用有理数：

```text
l=3, M=8, message m=3
I^-1 = [0,8), o_0=0
d^0 = 3, selected I^0=[3,5)

o_1=1/2
d^1 = 3 + ((3-3) + (1/2)*2) mod 2 = 4
selected I^1=[3,9/2)
midpoint(I^1)=15/4
midpoint(I^1)-d^1 = -1/4 in (-1/2,1/2]
```

因此 Algorithm 3 停止。但 Algorithm 4 得到

```text
y^0 = 3 + ((15/4-3) - (1/2)*2) mod 2 = 19/4
y^-1 = 19/4
round_half_down(19/4)=5 != 3.
```

错误发生在反向模旋转跨越 `[3,5)` 的边界时。模旋转保持圆周距离，但论文证明把它当成
保持实数轴上的普通距离；从 `-1/4` 变为 `-1/4+2=7/4`，所以局部 `1/2` 邻域不能直接
反推到初始整数的舍入单元。

### 正测度失败区域

上述反例不是连续随机偏移下的零概率单点。保持相同消息和 token 分割，令

```text
o_0 in [0,1/64],  o_1 in [15/32,17/32].
```

则

```text
d^0 = 3 + 8o_0 in [3,25/8] subset [3,5),
d^1 = 3 + 8o_0 + 2o_1 in [63/16,67/16] subset [3,9/2),
15/4-d^1 in [-7/16,-3/16] subset (-1/2,1/2].
```

反向第一步中点落在 `[75/16,77/16]`，再减去 `8o_0` 后落在
`[73/16,77/16]`，整个区间 round-half-down 均为 5。因此独立
`U[0,1)` 偏移落入该矩形的概率为

```text
(1/64) * (2/32) = 1/1024 > 0.
```

所以对这一固定消息和合法概率分割，论文停止规则具有正概率解码失败，不能以“几乎处处
正确”规避该反例。

运行 `python scripts/verify_rrc_theory.py` 可用 `Fraction` 精确重现，不涉及浮点误差。

## 5. 终止与恢复定理

### 定理 4：无跨切点时局部条件充分

逆旋转 `T_t^{-1}` 在 `o_t>0` 时以

```text
c_t = L^{t-1} + o_t Delta^{t-1}
```

为线性表示的切点，其两个连续分支为 `[L^{t-1},c_t)` 和 `[c_t,R^{t-1})`；`o_t=0` 时没有
内部切点。令最终普通误差 `e=y^T-d^T`。若：

1. `-1/2 < e <= 1/2`；
2. 对每个反向步骤 `t`，`d^t` 与 `d^t+e` 都位于 `T_t^{-1}` 的同一连续分支；

则完整反向重放得到 `y^{-1}=m+e`，从而 `round_half_down(y^{-1})=m`。

证明：在同一连续分支内，逆旋转只是减去同一常数，不发生加减父区间宽度，因此普通线性
误差保持为 `e`。对所有步骤反向归纳，最终误差仍为 `e`。条件 1 正好说明 `m+e` 落入整数
`m` 的 half-down 舍入单元 `(m-1/2,m+1/2]`。

第 4 节反例在第二次逆旋转时，`d^1=4` 位于切点右分支，而 `y^1=15/4` 位于左分支，恰好
违反条件 2。`no_wrap_recovery_certificate` 用精确有理数实现了这一充分条件，但生产编码器
使用完整反向重放检查，因为它更直接且不依赖该充分条件成立。

### 定理 5：Verified-RRC 条件精确恢复

假设：

1. 编码器和解码器对同一 token 前缀产生完全相同的概率分割；
2. 双方使用相同的初始区间、偏移序列和 round-half-down 规则；
3. 区间和旋转运算为精确算术；
4. 编码器仅在完整执行公开 Algorithm 4 后得到 `m` 时停止。

则接收方对发送的 token 序列执行 Algorithm 4，输出必为 `m`。

证明：发送方验证和接收方解码使用相同 token 序列。由假设 1，双方重建相同的
`I^{-1},...,I^T`；由假设 2，双方应用相同的逆映射序列；由引理 1，每一步逆旋转结果唯一，
故双方得到同一个 `y^{-1}`。发送方停止条件给出
`round_half_down(y^{-1})=m`，所以接收方输出 `m`。

该定理是 safety 结论：**返回成功结果时结果正确**。它本质上是运行时证书，不是有限终止
定理，也不是独立于实现一致性的形式化验证。

### 非定理：无条件有限终止

区间宽度满足

```text
Delta^T = M product_{t=0}^T p_{s_t}^{(t)}.
```

在存在 `rho<1` 且最终每步 `p_{s_t}^{(t)}<=rho` 等条件下，宽度趋于零。这足以使论文局部
中点条件最终成立，但不自动推出逆旋转后的中点进入消息 `m` 的 round-half-down 单元。
因此当前没有从“宽度趋零”到“Verified 条件必然触发”的完整证明。

实现设置 `max_tokens`，所以接口语义是部分算法：成功时由定理 5 保证正确；未触发时返回
`IncompleteEncodeError`，而不是伪造一个可解码结果。

## 6. Decimal 实现的条件对应

当前代码不是精确实数算法：

- provider 概率来自有限精度 softmax；
- 概率按 `probability_quantum` 量化并重归一化；
- 区间使用有限精度 Decimal；
- FP16/BF16/FP32、设备和 tokenizer 可能改变候选顺序或概率。

因此实现级正确性表述必须是：若双方得到相同量化分布、Decimal 运算未发生足以改变区间
选择或舍入结果的误差，并且 Verified 条件触发，则解码精确。当前
`decimal_precision = max(min_precision, integer_digits + guard_digits)` 是工程启发式，不构成
舍入误差上界。跨硬件和跨精度正确性必须靠实验验证，不能由理想算术定理直接推出。

若论文需要实现层面的严格保证，下一步应把概率先映射为公开整数频数，总频数固定为 `D`，
再用整数区间编码。这样编码器和解码器可以逐位复现同一分割，数值正确性可从“Decimal
误差足够小”升级为“整数运算不溢出且频数表一致”。这会改变实际目标分布，需要单独报告
整数频数量化引入的 `KL(R || Q)`。

## 7. 复杂度

设生成 `T` 个 tokens，第 `t` 步候选数为 `K_t`。

- 基础编码：`O(sum K_t)` 时间、`O(T)` 区间历史空间。
- Verified 检查：每次候选停止点需要 `O(t)` 逆重放。若检查次数为 `C`，附加时间为
  `O(C T)`；最坏情况下 `C=O(T)`，得到 `O(T^2)`。
- 解码：`O(sum K_t + T)` 时间、`O(T)` 空间。

Mock 实验中 Verified 检查平均只增加 0.04-0.15 token，但该经验结果不是复杂度上界。
